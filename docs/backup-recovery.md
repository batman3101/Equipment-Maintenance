# 백업 및 복구 전략

## 개요

이 문서는 CNC 설비 유지보수 앱의 데이터 백업 및 복구 전략에 대한 가이드라인을 제공합니다. 데이터 손실을 방지하고 시스템 장애 시 빠른 복구를 위한 절차를 설명합니다.

## 백업 전략

### 데이터베이스 백업

#### 자동 백업

Supabase는 다음과 같은 자동 백업을 제공합니다:

- **일일 백업**: 매일 자정(UTC)에 자동으로 수행됩니다.
- **주간 백업**: 매주 일요일에 수행되며, 4주간 보관됩니다.
- **월간 백업**: 매월 1일에 수행되며, 6개월간 보관됩니다.

#### 수동 백업

중요한 변경 사항이 있을 때는 수동 백업을 수행합니다:

```bash
# Supabase CLI를 사용한 수동 백업
supabase db dump -f backup-YYYY-MM-DD.sql
```

### 파일 스토리지 백업

Supabase Storage에 저장된 파일(이미지, 동영상 등)은 다음과 같이 백업합니다:

```bash
# Supabase Storage 백업
supabase storage download --bucket=breakdowns --output=./backups/storage/breakdowns
```

### 환경 설정 백업

환경 변수 및 설정 파일은 다음과 같이 백업합니다:

1. Vercel 프로젝트 설정 및 환경 변수를 JSON 형식으로 내보내기
2. Supabase 프로젝트 설정을 JSON 형식으로 내보내기
3. 백업 파일을 암호화하여 안전한 위치에 저장

## 복구 전략

### 데이터베이스 복구

#### Supabase 관리 콘솔을 통한 복구

1. Supabase 관리 콘솔에 로그인
2. 프로젝트 선택 > 데이터베이스 > 백업
3. 복구하려는 백업 지점 선택
4. "복원" 버튼 클릭

#### SQL 덤프 파일을 통한 복구

```bash
# SQL 덤프 파일을 사용한 복구
supabase db restore -f backup-YYYY-MM-DD.sql
```

### 파일 스토리지 복구

```bash
# Supabase Storage 복구
supabase storage upload --bucket=breakdowns --source=./backups/storage/breakdowns
```

### 장애 복구 절차

1. **장애 감지**: 서버 로그 및 모니터링 도구를 통한 장애 감지
2. **영향 평가**: 장애의 범위와 영향 평가
3. **복구 계획 수립**: 복구 방법 및 시간 계획 수립
4. **복구 실행**: 백업에서 데이터 복원
5. **검증**: 시스템 기능 및 데이터 무결성 검증
6. **문서화**: 장애 원인 및 복구 과정 문서화

## 재해 복구 계획

### 주요 시나리오

1. **데이터베이스 손상**: 가장 최근의 백업에서 복원
2. **스토리지 손실**: 백업된 파일에서 복원
3. **서비스 중단**: 대체 리전에 임시 배포

### 복구 시간 목표 (RTO)

- **중요 기능**: 4시간 이내
- **전체 시스템**: 24시간 이내

### 복구 지점 목표 (RPO)

- 최대 24시간 이내의 데이터 손실 허용

## 테스트 및 유지 관리

### 백업 테스트

- 분기별로 백업에서 테스트 환경으로 복원 테스트 수행
- 복원된 데이터의 무결성 검증

### 문서 유지 관리

- 백업 및 복구 절차 문서를 분기별로 검토 및 업데이트
- 새로운 기능이나 데이터 구조가 추가될 때마다 문서 업데이트
